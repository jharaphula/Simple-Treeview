Use Case Name

Transact Sale

Brief Description

This use case describes how a Sales Clerk uses the Point of Sale terminal to process electronic-funds-transfer sales.

Pre-conditions

Terminal is registered

A Style example for indentation

The system is currently registered with the Bank Host. (See Register Terminal use case.)

Post-conditions

Alternative Flows

Invalid PIN

if at {Transaction denied} (in Basic Flow) the Bank Host responds that the PIN is invalid

        The system prints "transaction denied" on the customer's receipt.

        The system informs the Sales Clerk and the Customer that the PIN was invalid. The system informs the Sales Clerk and the Customer that the PIN was invalid. The system informs the Sales Clerk and the Customer that the PIN was invalid.

        The system prompts the Customer to reenter their PIN.

        The Customer reenters their PIN.

        The use case resumes at {Reprocess transaction} (in Basic Flow)

Insufficient Funds

if at {Transaction denied} (in Basic Flow) the Bank Host responds that there are insufficient funds in the customer's account         The system prints "transaction denied" on the receipt.

        The system informs the Sales Clerk and the Customer that there are insufficient funds in the account.

        The use case ends.

 

Daily Transaction Limit Exceeded

    If at {Transaction denied} (in Basic Flow) the Bank Host responds that the customer has exceeded their daily withdrawal limit on the nominated account

        The system prints "transaction denied" on the receipt.

        The system informs the Sales Clerk and the Customer that the daily limit has been exceeded.

        The use case ends.

Fraud Alert On Transaction Card

    If at {Transaction denied} (in Basic Flow) the Bank Host responds that there is a fraud alert on the transaction card.

        The system prints "transaction denied" on the receipt.

        The system instructs the Sales Clerk to hold the transaction card and contact the bank immediately.

        The system informs the Customer that the transaction has been denied.

        The use case ends.

Special Requirements

Transaction Encryption

    All communication between the system, the PIN pad and the Bank Host shall be encrypted using 3DES
